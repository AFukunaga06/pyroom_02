# JANコード管理システム ユーザーマニュアル

## 概要

JANコード管理システムは、商品検品・在庫管理・伝票作成などの業務を効率化するためのデスクトップアプリケーションです。

## 主な機能

### 1. JANコード座標取得
画面上の任意の位置をクリックして、座標情報を取得します。

**使用方法:**
1. 「座標取得」ボタンをクリック
2. 画面上の目的の位置をクリック
3. 座標が自動的に `input.txt` に記録されます

### 2. JANコードコピー
取得した座標データをクリップボードに転送します。

**使用方法:**
1. 「JANコード等のコピー」ボタンをクリック
2. サブアプリケーションが起動します
3. JANコードを入力してコピーボタンをクリック

### 3. 商品情報抽出
JANコードから商品情報を自動取得します。

**使用方法:**
1. `input.txt` にJANコードを入力
2. 「商品抽出」ボタンをクリック
3. 商品情報が `output.txt` に出力されます

### 4. シートチェック
Google Sheetsのデータをチェックし、重複や欠損を検出します。

**使用方法:**
1. 「重複と項目抜けのチェック」ボタンをクリック
2. チェック結果が画面に表示されます
3. 詳細は `checkd01.txt` と `checkd02.txt` に保存されます

### 5. 一括実行
上記の機能をワンクリックで順次実行します。

**使用方法:**
1. 「一括実行」ボタンをクリック
2. 全ての処理が自動的に実行されます
3. 進捗状況が画面に表示されます

## ボタンの説明

### メイン機能ボタン（赤色）
- **重複と項目抜けのチェック**: データの整合性をチェック

### サブアプリケーションボタン（緑色）
- **JANコード等のコピー**: JANコード操作サブアプリを起動
- **廃番処理サブフォーム**: 廃番商品登録サブアプリを起動

### 実行ボタン（オレンジ色）
- **貼り付けて実行**: クリップボードデータを処理

### 外部リンクボタン（青色）
- **チェックシート**: Google Sheetsを開く
- **商品n**: 商品データベースを開く
- **藤原産業**: 藤原産業のWebサイトを開く

### バッチ実行ボタン（青色）
- **Type1**: Type1バッチファイルを実行
- **Type2**: Type2バッチファイルを実行

### ファイル操作ボタン（青色）
- **input.txt**: 入力ファイルを開く
- **output.txt**: 出力ファイルを開く
- **checkd01.txt**: チェック結果ファイル1を開く
- **checkd02.txt**: チェック結果ファイル2を開く

### 特殊機能ボタン（紫色）
- **座標取得**: マウス座標を取得
- **一括実行**: 全機能を順次実行

## ファイル形式

### input.txt
```
JANコード	1234567890123
ブランド名	サンプルブランド
商品名	サンプル商品
```

### output.txt
```
JANコード,商品名,価格,取得日時
1234567890123,サンプル商品,1000,2025-01-02 10:30:00
```

### checkd01.txt
重複チェック結果
```
重複JANコード: 1234567890123
行番号: 5, 10
```

### checkd02.txt
欠損チェック結果
```
欠損フィールド: 商品名
行番号: 3
```

## エラー対処法

### よくあるエラーと解決方法

#### 1. ファイルが見つからないエラー
**症状**: 「ファイルが存在しません」というエラーが表示される
**解決方法**: 
- 必要なファイル（input.txt等）がプロジェクトフォルダに存在するか確認
- ファイル名に誤字がないか確認

#### 2. Google Sheets接続エラー
**症状**: 「Google Sheetsに接続できません」というエラーが表示される
**解決方法**:
- インターネット接続を確認
- Google API認証情報ファイルが正しく配置されているか確認
- Google Sheetsの共有設定を確認

#### 3. 座標取得エラー
**症状**: 座標取得が正常に動作しない
**解決方法**:
- アプリケーションを管理者権限で実行
- セキュリティソフトの設定を確認
- 他のアプリケーションとの競合を確認

#### 4. 文字化けエラー
**症状**: 日本語が正しく表示されない
**解決方法**:
- システムの言語設定を確認
- フォントの設定を確認
- ファイルのエンコーディングを確認（UTF-8）

## 設定のカスタマイズ

### config.json の編集

```json
{
  "api_key": "your_api_key",
  "input_path": "input.txt",
  "output_path": "output.txt",
  "checkd01_path": "checkd01.txt",
  "checkd02_path": "checkd02.txt",
  "google_credentials_path": "samplep20240906-5ae36c9a4acd.json",
  "spreadsheet_id": "17Le1KA9nzMREt0Qp9_elM1OF1q8aSp-GDBZRPOntNI8",
  "log_level": "INFO",
  "max_retries": 3,
  "timeout": 30
}
```

### 設定項目の説明

- **api_key**: 外部API呼び出し用のキー
- **input_path**: 入力ファイルのパス
- **output_path**: 出力ファイルのパス
- **checkd01_path**: チェック結果ファイル1のパス
- **checkd02_path**: チェック結果ファイル2のパス
- **google_credentials_path**: Google API認証情報ファイルのパス
- **spreadsheet_id**: Google SheetsのスプレッドシートID
- **log_level**: ログレベル（DEBUG, INFO, WARNING, ERROR）
- **max_retries**: API呼び出しの最大リトライ回数
- **timeout**: API呼び出しのタイムアウト時間（秒）

## パフォーマンス向上のヒント

### 1. 大量データの処理
- 一度に処理するデータ量を制限する
- 処理中は他のアプリケーションを最小限に抑える

### 2. Google Sheets API の効率的な使用
- 不要なAPI呼び出しを避ける
- バッチ処理を活用する

### 3. メモリ使用量の最適化
- 定期的にアプリケーションを再起動する
- 不要なファイルを削除する

## セキュリティ注意事項

### 1. 認証情報の保護
- Google API認証情報ファイルを適切に保護する
- 認証情報を他人と共有しない

### 2. データの保護
- 重要なデータファイルを暗号化する
- アクセス権限を適切に設定する
